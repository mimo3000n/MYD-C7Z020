# 🕹️ Controlling the Zynq-7000 PS via OpenOCD

This document explains how to take direct control of the **ARM Cortex-A9** cores on the MYD-C7Z020 using OpenOCD. This allows for "Bare Metal" interaction, bypassing the Linux Kernel and U-Boot entirely.

## 1. Understanding the JTAG Chain
The Zynq-7000 is a multi-tap device. When you connect via JTAG, OpenOCD sees two main components:
1.  **PL TAP:** The FPGA logic (Boundary Scan).
2.  **PS DAP:** The Debug Access Port for the ARM cores.



---

## 2. Configuration for PS Access

Use the `zynq_hs3.cfg` we created earlier, but ensure the `target/zynq_7000.cfg` is sourced. This file (provided by OpenOCD) defines the two CPU targets as `zynq.cpu0` and `zynq.cpu1`.

**To start the controller:**
```bash
openocd -f zynq_hs3.cfg
```

---

## 3. Core Manipulation (Telnet)

Connect to the OpenOCD console (`telnet localhost 4444`) to issue hardware commands:

### Basic CPU Control
* **List Targets:** `targets` (Shows the status of both A9 cores).
* **Halt Core 0:** `targets zynq.cpu0; halt` (Freezes the processor mid-instruction).
* **Resume Execution:** `resume`
* **Soft Reset:** `reset init` (Resets the processor and runs any initialization scripts).

### Inspecting CPU State
When a core is halted, you can look inside the "brain" of the ARM processor:
* **View Registers:** `reg` (Shows R0-R15, CPSR, and the Floating Point registers).
* **Modify a Register:** `reg r0 0xDEADBEEF` (Manually injects data into the CPU).

---

## 4. Direct Memory Access (The AXI Backbone)

Because the PS is the master of the system, you can use it to reach any address on the AXI bus, including DDR, OCM, and FPGA IP cores.

| Command | Action | Example |
| :--- | :--- | :--- |
| **mdw** | Read 32-bit Word | `mdw 0xF8000000` (Read SLCR Lock) |
| **mww** | Write 32-bit Word | `mww 0xF8000008 0xDF0D` (Unlock SLCR) |
| **dump_image**| Save RAM to File | `dump_image ram_dump.bin 0x0 0x10000` |
| **load_image**| Load File to RAM | `load_image my_code.bin 0x00100000` |



---

## 5. Initializing DDR RAM via OpenOCD

If you boot the board in **JTAG Mode (SW1=0000)**, the DDR RAM is **OFF**. You cannot write to addresses above `0x00100000` until the DDR controller is initialized.

You can do this without PetaLinux by using the `ps7_init.tcl` script generated by Vivado:
1.  Inside OpenOCD/Telnet:
    ```tcl
    source ps7_init.tcl
    ps7_init
    ps7_post_config
    ```
2.  Now the DDR3 on your MYIR board is "alive," and you can load large files into it.

---

## 6. Interaction Comparison: OpenOCD vs. XSCT

| Feature | OpenOCD | Xilinx XSCT |
| :--- | :--- | :--- |
| **Drivers** | Open Source (FTDI) | Proprietary (Xilinx) |
| **Speed** | Fast / Lightweight | Slower / Heavy |
| **GDB Support** | Excellent (Native) | Proprietary Interface |
| **Scripting** | Tcl (Standard) | Tcl (Custom AMD commands) |

---

## 🛠 Troubleshooting PS Control
* **"Target not halted":** You cannot read/write memory while the CPU is running. Always run `halt` first.
* **"Sticky Error" on DAP:** This usually means you tried to access a memory address that doesn't exist or is currently powered down (like a PL IP core with no clock).
* **Security Violation:** If the "TrustZone" bits are set, you may be able to halt the CPU but unable to read "Secure" memory regions.
